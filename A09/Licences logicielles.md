
# Définition

Une licence logicielle est un contrat légal qui indique comment un logiciel peut être utilisé. Elle précise ce que l'utilisateur a le droit de faire et ce qui est interdit. La licence est définie par le propriétaire du logiciel.

Elle peut contenir notamment
- Comment installer et utiliser le logiciel
- Si l'on peut en faire des copies
- Si l'on peut le partager ou le redistribuer
- Si les mises à jour sont autorisées ou payantes
- La durée d'utilisation
- Les restrictions
- Les responsabilités de l'auteur et du fournisseur

Il est important de lire la licence avant d'installer un logiciel, car le non respect de certaines règles peut avoir des conséquences juridiques


# Types de licences

Chaque types de licences logicielles défini un cadre d'utilisation, de partage et de distribution différent.

- **Licence propriétaire** : Propriété exclusive à l'éditeur, l'utilisateur ne peut ni la modifier ni la redistribuer.
- **Licence d'évaluation** : Version temporaire, il faut acheter la licence après la période d'essai.
- **Licence commerciale** : Licence payante, avec règles d'utilisation strictes.
- **Licence en volume** : Pour les organisations, permet des installations multiples à prix réduits.
- **Licence cloud** : Logiciel utilisé en ligne, abonnement mensuel ou annuel.
- **Licence académique** : Réservée aux étudiants, tarifs réduits pour un usage éducatif.
- **Licence gratuite** : Logiciel gratuit, mais conditions à respecter, propriétaire ou open source.
- **Licence libre de droits** : Paiement unique, utilisation perpétuelle.
- **Licence libre** : Utilisation, modification et redistribution libre.
- **Licence open source** : Le code source est accessible et modifiable, encore la contribution de la communauté et la transparence.

# FOSS

FOSS signifie *Free and Open Source Software*. Il désigne des logiciels dont le code source est accessible et modifiable.

Les utilisateurs peuvent :
- Utiliser le logiciel librement
- Accéder au code source
- Le modifier selon leur besoins
- Le redistribuer
- Contribuer à son amélioration

L'idée principale est de favoriser la liberté, la transparence et la collaboration dans le développement logiciel.


# Copyleft

Le copyleft est un principe de licences libres qui oblige à garder un logiciel et ses versions modifiées toujours libres.

- **Licences copyleft (ex : GPL)** : Si le code est modifié ou redistribué, il doit être publié sous la même licence --> toutes les versions restent libres.
- **Licences non copyleft (ex : Apache, MIT)** : Le code est utilisable et modifiable, sans obligation de le publier même dans des logiciels propriétaires.


# MIT License

La licence MIT est une licence permissive, courte et simple. Elle autorise l'utilisation, la modification et la redistribution du logiciel, à condition de préserver les mentions de copyright et de licence. 

Les œuvres dérivées peuvent être redistribuées sous n'importe quelle licence, avec ou sans le code source.
